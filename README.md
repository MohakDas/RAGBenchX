# RAGBenchX

RAGBenchX is a document-driven benchmarking framework designed to evaluate and compare Retrieval-Augmented Generation (RAG) pipelines on user-provided documents. The system ingests PDF or text documents, performs preprocessing through configurable chunking strategies, and converts document segments into dense vector embeddings using the sentence-transformers all-MiniLM-L6-v2 model. These embeddings are indexed using FAISS to enable efficient similarity-based retrieval. The framework implements two RAG pipelines that differ in chunking configurations, specifically using chunk sizes of 512 and 1024 tokens, allowing analysis of how context granularity impacts retrieval quality and answer generation. For evaluation, synthetic question–answer pairs are automatically generated from the document using a locally hosted language model served through Ollama (e.g., qwen2:1.5b). Each evaluation query is embedded and used to retrieve the top-k most relevant document chunks from the FAISS index, which are then provided as contextual input to the language model to generate answers. The generated responses from each pipeline are compared against the reference answers using cosine similarity between embedding representations to estimate semantic alignment, while inference latency is recorded to measure performance. The benchmarking results are aggregated and visualized through an interactive Streamlit dashboard, enabling systematic experimentation with chunk sizes, retrieval strategies, and RAG pipeline configurations.

TL;DR - RAGBenchX benchmarks different Retrieval-Augmented Generation pipelines by automatically generating evaluation questions from uploaded documents and comparing performance across similarity and latency metrics using a visual dashboard.
